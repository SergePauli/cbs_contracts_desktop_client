# cbs_contracts_desktop_client

Нативный Windows-клиент для системы "База контрактов и контрагентов", реализуемый на WinUI 3 и Windows App SDK с опорой на существующий web-клиент `cbs_contracts_webclient`.

## Текущий статус

- Готов базовый WinUI 3 проект для Visual Studio 2022.
- Настроены `CbsContractsDesktopClient.csproj`, `CbsContractsDesktopClient.sln` и структура `src/`.
- Приложение запускается как desktop-клиент с окном, адаптированным под Full HD без выхода под taskbar.
- Подключены DI и `HttpClient`.
- Работает авторизация против внешнего API.
- Реализовано сохранение логина и пароля через Windows Credential Manager.
- Собрана и стилизована первая форма: экран авторизации.
- После успешного входа открывается базовый `AppShell` с layout-регионами.
- Настроен `.gitignore`, проект очищен от временных файлов и лишних asset-файлов.

## Что уже сделано

### Инфраструктура

- Создан проект WinUI 3 под .NET 8.
- Подключены:
  - `Microsoft.WindowsAppSDK`
  - `CommunityToolkit.Mvvm`
  - `Microsoft.Extensions.DependencyInjection`
  - `Microsoft.Extensions.Http`
- В `App.xaml.cs` настроен контейнер зависимостей.
- В `MainWindow.xaml.cs` настроены:
  - размер и центрирование окна
  - учет рабочей области экрана Windows
  - иконка приложения

### Авторизация

- Реализован `AuthService` для входа через API.
- Базовый адрес API сейчас настроен на `http://serge-lenovo:5000`.
- Вход выполняется через endpoint `POST auth/login`.
- Тело запроса соответствует web-клиенту: `name` + `password`.
- Ответ сервера распознается и преобразуется в локальную модель пользователя.
- Текущий пользователь сохраняется в `UserService`.

### Хранение учетных данных

- Реализован `CredentialManagerService`.
- Доступен флаг `Запомнить меня`.
- При запуске логин и пароль автоматически подставляются из системного хранилища Windows.

### Экран логина

- Собран экран авторизации на XAML.
- Добавлены:
  - поля логина и пароля
  - индикатор загрузки
  - вывод ошибки входа
  - чекбокс `Запомнить меня`
  - ссылки `Регистрация` и `Не помню пароль` с временными заглушками
- Правая колонка оформлена как информационная панель с логотипом, названием системы и версией клиента.

### Shell после логина

- Добавлен `AppShellPage` как базовый каркас приложения после входа.
- Layout разделен на 5 регионов:
  - левый sidebar
  - верхний top bar
  - центральная область контента
  - правый audit panel
  - нижний footer bar
- Регионы вынесены в отдельные `UserControl`:
  - `NavigationSidebarView`
  - `TopBarView`
  - `ContentHostView`
  - `AuditPanelView`
  - `FooterBarView`
- Визуальный стиль shell приведен ближе к нативному WinUI / Windows 11, без тяжелых web-границ.

### Git и структура проекта

- Добавлен `.gitignore` под Visual Studio / .NET / WinUI.
- Оставлен актуальный solution-файл, созданный Visual Studio.
- В `Assets/` оставлены только реально используемые ресурсы.

## Структура проекта

- `src/Models/` - модели данных
- `src/Services/` - API, пользовательское состояние, системные сервисы
- `src/ViewModels/` - MVVM-логика
- `src/Views/` - XAML-страницы и code-behind
- `Assets/` - используемые иконки и изображения
- `docs/` - сопроводительная документация

## Что остается временным

- Центральная область `AppShell` пока показывает только статический placeholder вместо реальной страницы.
- Правая audit-панель пока является визуальной заглушкой без живых данных.
- Ссылки `Регистрация` и `Не помню пароль` пока ведут только на заглушки.

## Запуск в Visual Studio 2022

1. Открыть `CbsContractsDesktopClient.sln`.
2. Выбрать конфигурацию `Debug | x64`.
3. Убедиться, что установлены workload'ы для .NET desktop development и компоненты WinUI 3 / Windows App SDK.
4. Запустить проект через `F5`.
