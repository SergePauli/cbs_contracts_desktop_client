# Окно авторизации в WinUI 3

## Обзор

Создано окно авторизации с нативным WinUI 3 UI, визуально похожее на веб-версию. Реализует:

- Форму входа с полями username/password
- Валидацию и обработку ошибок
- Сохранение пользователя в UserService (аналог MobX store)
- Подготовку для ролевой навигации

## Созданные файлы

- `Views/LoginPage.xaml` — XAML разметка окна
- `Views/LoginPage.xaml.cs` — код-behind страницы
- `ViewModels/LoginViewModel.cs` — ViewModel с логикой авторизации
- `Models/User.cs` — модели пользователя и DTO
- `Services/AuthService.cs` — сервис для HTTP авторизации
- `Services/UserService.cs` — сервис состояния пользователя

## Следующие шаги

1. **Настроить DI контейнер** в `App.xaml.cs` для регистрации сервисов
2. **Добавить навигацию** — после успешного логина переходить к MainPage
3. **Реализовать MainPage** с меню и ролевой навигацией
4. **Подключить реальный API** — заменить заглушку в AuthService на endpoint из веб-клиента
5. **Добавить обработку ролей** — в зависимости от роли пользователя показывать разные разделы

## Как интегрировать в MainWindow

В `MainWindow.xaml.cs` добавить логику:

- Показывать LoginPage при запуске
- После логина заменять контент на MainPage
- Обрабатывать logout для возврата к авторизации

## Тестирование

- Запустить приложение — должно открыться окно авторизации
- Ввести любые данные — сработает заглушка (1 сек задержка)
- После "успешного" входа добавить навигацию к главному экрану

## Зависимости

Убедитесь, что установлены NuGet пакеты:

- `CommunityToolkit.Mvvm`
- `Microsoft.Extensions.DependencyInjection`
- `Newtonsoft.Json` (или `System.Text.Json`)
